Why should you use ELK ?

    Data constantly flows into your systems, but it can quickly grow to be fat !
    As your data set grows larger, your analytics will slow up, resulting in sluggish insights.
  • How can you maintain valuable business insights?
  • Use ELK !
  1. Centralize your logs
  2. Parse and gather only the fields you need
  3. Visualize your logs and identify the information quickly

The components

  1. Logstash: Processes incoming logs

  2. Elasticsearch: Stores all of the logs

  3. Kibana: Web interface for searching and visualizing logs

Logstash

  • Centralize data processing of all types
  • Normalize varying schema and formats
  • Quickly extend to custom log formats
  • Easily add plugins for custom data sources
Source

Elasticsearch

  • Distributed, scalable, and highly available
  • Real-time search and analytics capabilities
  • Sophisticated RESTful API
Source

Kibana

  • Flexible analytics and visualization platform
  • Real-time summary and charting of streaming data
  • Intuitive interface for a variety of users
  • Instant sharing and embedding of dashboards
Source
